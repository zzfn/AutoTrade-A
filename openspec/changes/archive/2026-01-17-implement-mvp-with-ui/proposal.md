# 提案: 实现 MVP 及 Web UI

## ID

`implement-mvp-with-ui`

## 背景

项目需要一个“最小可验证版本”(MVP)，具备以下功能：

1.  运行一个简单的交易策略（非 Qlib）。
2.  实时交易执行（连接 Alpaca）。
3.  回测功能。
4.  通过用户界面 (UI) 可视化这些操作。

目前项目仅有基本结构，缺乏应用逻辑和 UI。

## 目标

1.  **简单策略**: 使用 `LumiBot` 实现一个基础策略（如简单的动量或趋势跟随）。
2.  **Web 服务**: 使用 FastAPI 构建后端，并直接提供前端页面服务（无需独立的 Frontend 项目）。
3.  **Web UI**: 构建一个现代、美观的前端界面（HTML/CSS/JS + Jinja2 模板），用于展示实时信号/订单和控制回测。
4.  **集成**: 确保 `make run` 可以启动完整的系统。

## 范围

- **范围内**:
  - LumiBot 策略实现（基础版）。
  - FastAPI 服务器设置 (`uvicorn`)。
  - 前端实现：使用 FastAPI 的 `Jinja2Templates` 和静态文件 (`static`)。
  - 实时仪表盘页面。
  - 回测配置与结果页面。
  - Alpaca 集成（模拟盘/Paper Trading）。
- **范围外**:
  - Qlib 策略集成（推迟）。
  - 复杂的用户认证（假设单用户）。
  - Next.js 或其他复杂的前端框架。

## 风险

- **LumiBot 并发**: 在 Web 服务器（异步）旁运行 LumiBot（阻塞/线程）需要小心的线程/多进程设计。
- **UI 响应性**: 使用简单的轮询或 WebSocket 确保 UI 能够低延迟地反映实时状态。
